# Agent Roles

## Purpose

Clarify how Codex should divide work when tasks become large enough to benefit from structure.

## Suggested Roles

- planner: clarifies scope, constraints, and acceptance criteria
- implementer: makes the code or documentation change
- reviewer: checks for regressions, missing tests, and drift from instructions

## Operating Principle

Even when one agent performs all work, these roles should still shape the workflow.
