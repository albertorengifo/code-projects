# Prompting Patterns

## Default Structure

Prompts should usually include:

- `Goal`
- `Context`
- `Constraints`
- `Done when`

## Recommended Habits

- mention concrete paths instead of vague areas
- define what completion looks like
- constrain the blast radius of changes
- ask for planning first when architecture is still in motion

## Anti-Patterns

- asking for broad implementation without file or scope hints
- mixing architecture decisions with deep implementation in one step
- leaving verification undefined
